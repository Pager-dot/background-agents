## Quick Bootstrap

From the repository root, run the setup script to install everything at once:

```bash
bash .openinspect/setup.sh
```

This installs JS dependencies, builds `@open-inspect/shared`, installs git hooks, and sets up the Python environment for `packages/modal-infra` when possible.

## Run the Web App Locally

Connect your local UI to a deployed control plane and Modal backend.

### 1. Create env file

```bash
cp packages/web/.env.example packages/web/.env.local
```

### 2. Fill required variables

```bash
# GitHub App OAuth
GITHUB_CLIENT_ID=your_github_app_client_id
GITHUB_CLIENT_SECRET=your_github_app_client_secret

# NextAuth
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=<run: openssl rand -base64 32>

# Deployed backend
CONTROL_PLANE_URL=https://open-inspect-control-plane-<name>.<subdomain>.workers.dev
NEXT_PUBLIC_WS_URL=wss://open-inspect-control-plane-<name>.<subdomain>.workers.dev

# Must match the control-plane INTERNAL_CALLBACK_SECRET
INTERNAL_CALLBACK_SECRET=your_shared_secret

# Optional access control
ALLOWED_USERS=
ALLOWED_EMAIL_DOMAINS=
```

<Warning>
  If you're connecting to someone else's deployed backend, use *their* `INTERNAL_CALLBACK_SECRET` value — do not generate a new one.
</Warning>

### 3. Add localhost callback to GitHub App

In your GitHub App settings → Callback URLs, add:

```
http://localhost:3000/api/auth/callback/github
```

### 4. Start the dev server

```bash
npm run dev -w @open-inspect/web
```

Open [http://localhost:3000](http://localhost:3000), sign in with GitHub, and create a session.

## TypeScript Development

### Build order

Always build `@open-inspect/shared` first — other packages import from it at build time:

```bash
npm run build -w @open-inspect/shared
```

### Common commands

```bash
# Build everything
npm run build

# Build specific packages
npm run build -w @open-inspect/control-plane

# Lint and format
npm run lint:fix
npm run format

# Type check all packages
npm run typecheck
```

### Running tests

```bash
# All TypeScript packages
npm test

# Specific packages
npm test -w @open-inspect/control-plane
npm test -w @open-inspect/web
npm test -w @open-inspect/github-bot
npm test -w @open-inspect/slack-bot
npm test -w @open-inspect/linear-bot

# Control plane integration tests (real workerd + D1)
npm run test:integration -w @open-inspect/control-plane
```

<Note>
  Control plane integration tests run inside a real `workerd` runtime with Miniflare, using real D1 bindings. They use `isolatedStorage: false` and require cleanup via `cleanD1Tables()` in `beforeEach`.
</Note>

## Python Development (modal-infra)

```bash
cd packages/modal-infra

# Install dependencies (preferred)
uv sync --frozen --extra dev

# Alternative
pip install -e ../sandbox-runtime
pip install -e ".[dev]"

# Run tests
pytest tests/ -v

# Lint and format
ruff check --fix
ruff format
```

## Common Issues

<AccordionGroup>
  <Accordion title="OAuth error: redirect_uri is not associated with this application">
    Your GitHub callback URL doesn't exactly match the running app URL. Add `http://localhost:3000/api/auth/callback/github` to your GitHub App's callback URLs.
  </Accordion>
  <Accordion title="Access denied after sign-in">
    Check `ALLOWED_USERS` and `ALLOWED_EMAIL_DOMAINS` in `packages/web/.env.local`. If both are empty, access is denied by default unless `UNSAFE_ALLOW_ALL_USERS=true` is set.
  </Accordion>
  <Accordion title="Session APIs return 401">
    `INTERNAL_CALLBACK_SECRET` in your `.env.local` doesn't match the deployed control plane's secret.
  </Accordion>
  <Accordion title="WebSocket disconnects immediately">
    For a deployed control plane use `wss://...`; for a local control plane use `ws://...`.
  </Accordion>
  <Accordion title="Prompts queue but no sandbox work happens">
    The control plane cannot reach Modal (or Modal is not properly configured/deployed).
  </Accordion>
</AccordionGroup>
