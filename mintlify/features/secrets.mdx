Open-Inspect lets you store environment variables — API keys, database URLs, credentials — and inject them into every sandbox automatically. Secrets are encrypted at rest and never exposed to the browser (only key names are visible in the UI).

---

## Quick Start

<Steps>
  <Step title="Open Settings">
    Go to your Open-Inspect web app and navigate to **Settings**. The **Secrets** tab is selected by default.
  </Step>
  <Step title="Choose scope">
    Use the scope dropdown to choose **All Repositories (Global)** or a specific repository.
  </Step>
  <Step title="Add a secret">
    Click **Add secret**, enter a key and value, then click **Save**.
  </Step>
</Steps>

The next sandbox you launch will have the secret available as an environment variable.

---

## Global vs. Repository Secrets

| Scope | Applies To | Use Case |
|-------|-----------|---------|
| **Global** | All repositories | API keys shared across projects (`ANTHROPIC_API_KEY`, `DATABASE_URL`) |
| **Repository** | One specific repo | Repo-specific credentials (`STRIPE_SECRET_KEY`, `AWS_ACCESS_KEY_ID`) |

**Precedence**: Repository secrets override global secrets with the same key. When viewing a repository's secrets, inherited global keys are shown read-only with a "Global" badge. If you override a global key at repo level, the global entry shows "(overridden by repo)."

## Adding and Managing Secrets

### Add from Settings

1. Go to **Settings > Secrets**
2. Select a scope (global or specific repository)
3. Click **Add secret**, enter key name and value, then **Save**

Keys are automatically uppercased when saved. `my_api_key` becomes `MY_API_KEY`.

### Paste a `.env` file

Paste a `.env`-formatted block (e.g., `KEY=value`) into any input field — Open-Inspect parses it automatically and populates multiple rows.

### Update a secret

Existing values are masked (`••••••••`). Type a new value and click **Save** to update. Leave empty to keep the current value.

### Delete a secret

Click the delete button next to any secret row and confirm.

## Common Examples

| Key | Scope | Purpose |
|-----|-------|---------|
| `ANTHROPIC_API_KEY` | Global | Claude API access *(required for Daytona provider)* |
| `OPENAI_OAUTH_REFRESH_TOKEN` | Repo | OpenAI Codex access |
| `OPENAI_OAUTH_ACCOUNT_ID` | Repo | OpenAI Codex access |
| `DATABASE_URL` | Repo | Database connection string |
| `AWS_ACCESS_KEY_ID` | Repo | AWS credentials |
| `STRIPE_SECRET_KEY` | Repo | Stripe API key |

## Reserved Keys

The following keys are reserved for system use and cannot be set as secrets:

`PYTHONUNBUFFERED`, `SANDBOX_ID`, `CONTROL_PLANE_URL`, `SANDBOX_AUTH_TOKEN`, `REPO_OWNER`, `REPO_NAME`, `GITHUB_APP_TOKEN`, `SESSION_CONFIG`, `RESTORED_FROM_SNAPSHOT`, `OPENCODE_CONFIG_CONTENT`, `PATH`, `HOME`, `USER`, `SHELL`, `TERM`, `PWD`, `LANG`

The UI shows a validation error if you attempt to save a reserved key.

---

## Security

- Secrets are encrypted with **AES-256-GCM** before being stored in the database
- Values are **never returned by the API** after saving — only key names are visible
- Secrets are decrypted at sandbox creation time and injected as environment variables
- System variables (set by the control plane) always take precedence over user-defined secrets

## Limits

| Constraint | Limit |
|-----------|-------|
| Max secrets per scope | 50 |
| Max key length | 256 characters |
| Max value size | 16 KB |
| Max total value size (per scope) | 64 KB |
| Key format | `[A-Za-z_][A-Za-z0-9_]*` |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Model not found errors (Daytona provider)">
    If using `sandbox_provider = "daytona"` with Claude models and seeing "Model not found" errors, add `ANTHROPIC_API_KEY` as a global secret. Unlike Modal, Daytona does not inject LLM API keys automatically.
  </Accordion>
  <Accordion title="Secret not appearing in sandbox">
    1. Verify the secret is saved under the correct scope
    2. Check the key isn't in the reserved keys list
    3. New secrets only apply to **new** sandboxes — restart your session to pick up changes
  </Accordion>
  <Accordion title="Key name was auto-changed">
    Keys are automatically uppercased when saved. `my_api_key` becomes `MY_API_KEY`.
  </Accordion>
</AccordionGroup>
