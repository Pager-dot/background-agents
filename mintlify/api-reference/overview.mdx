## Base URL

The control plane is a Cloudflare Worker. All API requests go to:

```
https://open-inspect-control-plane-{deployment_name}.{subdomain}.workers.dev
```

Replace `{deployment_name}` with your Terraform `deployment_name` and `{subdomain}` with your Cloudflare Workers subdomain.

---

## Authentication

All endpoints except `/health` require authentication via a session cookie from GitHub OAuth.

The web app handles authentication automatically. For direct API calls, include the session cookie obtained after signing in:

```bash
curl -H "Cookie: next-auth.session-token=..." \
  https://open-inspect-control-plane-{deployment_name}.{subdomain}.workers.dev/sessions
```

WebSocket connections require a short-lived token — see [WebSocket Protocol](/api-reference/websocket).

## Common Response Codes

| Code | Meaning |
|------|---------|
| `200` | Success |
| `201` | Created |
| `400` | Bad request / invalid input |
| `401` | Unauthenticated |
| `403` | Forbidden |
| `404` | Resource not found |
| `409` | Conflict (e.g., concurrent run active) |
| `413` | Payload too large |
| `500` | Internal server error |

---

## Pagination

List endpoints support cursor-based pagination via `?cursor=` and `?limit=` query parameters. Responses include a `nextCursor` field when more results are available.

---

## Cross-Origin Requests

CORS headers are set on all responses. Only requests originating from the configured `WEB_APP_URL` are permitted for cookie-authenticated calls.
